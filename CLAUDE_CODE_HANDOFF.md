## 2. Architecture & Tech Stack

### Current architecture: Single HTML file

The entire site is **one `index.html` file** with inline CSS and JavaScript. No build step. No framework. No dependencies. No npm. No bundler.

```
ourvoicevote/
├── index.html               ← the site (now includes the "Today's News" section)
├── news.json                ← daily news data (GENERATED by the Action; never hand-edit)
├── scripts/
│   └── generate-news.mjs    ← daily news generator (Node 20, zero dependencies)
├── .github/workflows/
│   └── daily-news.yml        ← runs the generator ~9 AM Central, commits news.json
└── NEWS_ENGINE_SETUP.md      ← one-time setup (Anthropic API key, enabling Actions)
```

### Why it's this way

Ray edits via GitHub's web UI, not a code editor. The single-file approach means:
- No dev environment needed
- No build step
- No dependencies to break
- Netlify auto-deploys on every commit to `main`
- Anyone can maintain it

### External dependencies (loaded via CDN)

- **Google Fonts:** Playfair Display (display serif), Source Sans 3 (body sans), DM Serif Display (logo)
- That's it. No other external JS or CSS.

### Deployment flow

```
Edit index.html on GitHub → Commit to main → Netlify auto-deploys (30-60s) → Live
```

No CI/CD config needed. Netlify watches the repo's `main` branch.

## 5. JavaScript Functionality

All JS is inline at the bottom of `index.html`. Functions:

### Navigation
- Hamburger menu toggle (mobile)
- Smooth scroll for anchor links
- Close mobile menu on link click

### Scroll animations
- IntersectionObserver on `.fade-up` elements → adds `.visible` class at 10% threshold

### Find Your Representatives (`findReps()`)
- Takes address/zip from `#addressInput`
- Renders links to House.gov, Senate.gov, USA.gov, OpenStates (with encoded address)
- **Not a real API call** — just generates links to official lookup tools
- Enter key triggers search

### Search Bills (`searchBills()`)
- Takes query from `#billSearch`
- Renders links to Congress.gov, GovTrack, LegiScan (with encoded query)
- **Not a real API call** — just generates search links
- Enter key triggers search

### Donations (`selectTier()`, `clearTierSelection()`, `handleDonate()`)
- Tier selection: $5/$15/$25/$50 with visual selection state
- Custom amount input clears tier selection
- `handleDonate()` opens PayPal.me link with amount in new tab

## 6. Hard Rules — DO NOT VIOLATE

These are non-negotiable. Breaking any of these is a showstopper.

### Content rules
- ✅ **Non-partisan.** No party affiliations. No candidate endorsements. Facts only.
- ✅ **Free only.** Every listed resource must be genuinely free. No paywalls. No paid tools.
- ✅ **Sourced.** Every statistic needs a cited source link.
- ✅ **No political donations.** Site explicitly states this for transparency.
- ✅ **Voter ID stance:** Supports photo ID requirements while noting free IDs are available.
- ✅ **Language assistance:** For U.S. citizens only.

### Technical rules
- ❌ **Never** add tracking pixels or third-party analytics without explicit owner approval.
- ❌ **Never** add affiliate links or paid resources.
- ❌ **Never** reproduce copyrighted material (song lyrics, full article paragraphs).
- ❌ **Never** quietly delete sections during an update — preserve everything unless explicitly told otherwise.
- ❌ **Never** add "how to set up donations" instructions to the public site.
- ❌ **Never** change the non-partisan stance, voter ID stance, or language assistance stance.

## 8.5 Daily News Engine (added June 9, 2026)

The **Today's News** section (`#news`, right after the hero) auto-updates every day
at ~9 AM Central. It is the only self-updating part of the site.

**Flow:** `daily-news.yml` (GitHub Actions cron, `0 14 * * *` UTC) → runs
`scripts/generate-news.mjs` → that script calls the Anthropic Messages API with the
server-side web search tool (`web_search_20250305`), gets a strict-JSON non-partisan
digest, validates it, and writes `news.json` → the Action commits `news.json` →
Netlify deploys. The browser fetches `news.json` and renders it.

**Key facts for maintainers:**
- **Do NOT hand-edit `news.json`** — it is overwritten daily. Edit the site by
  editing `index.html` / the generator.
- The render code lives in `index.html` (CSS `.news-*`, the `#news` markup, and the
  JS block beginning `/* ===== Today's News ===== */`).
- `news.json` schema: `{ updated, updatedLabel, national[], states{CC:[...]}, disclaimer }`.
  Items are `{ title, summary, source, url, date }`.
- Requires repo secret **`ANTHROPIC_API_KEY`**. Optional repo variable
  `ANTHROPIC_MODEL` (default `claude-opus-4-8`; `claude-haiku-4-5` is cheapest).
- **Fails safe:** if the API errors, JSON is bad, or too little is verifiable, the
  generator exits non-zero and `news.json` is left untouched — the site never breaks.
- States with no verified item that day show live-source links in the selector;
  a `fetch` failure shows national live-source links. So every state always "works."
- Non-partisan rules are enforced in the generator's system prompt AND in the
  validation step (unsourced items are dropped). This upholds Hard Rule #1.
- Time note: GitHub cron is UTC and ignores DST → `0 14` is 9 AM CDT / 8 AM CST.
  Use `0 15` for strict 9 AM in winter. Full setup: `NEWS_ENGINE_SETUP.md`.
